# 拾艺局运营后台 UI 设计图说明 v1

## 1. 设计目标

本稿为拾艺局运营后台 MVP 的低保真 UI 设计说明，用于：
- 明确页面结构
- 指导后续视觉设计
- 指导前端后台开发落地

设计原则：
- 信息优先，减少花哨装饰
- 列表、表单、详情三种页面结构统一
- 让运营人员 3 次点击内完成常见操作
- 风格简洁，偏专业后台，不抢艺术内容本身风头

## 2. 设计风格建议

- 整体风格：简洁、稳重、留白足够
- 主色：深灰黑 + 金色点缀
- 辅助色：白色、浅灰、功能色
- 字体风格：系统无衬线
- 圆角：中小圆角
- 阴影：轻阴影，避免厚重

建议色值：
- 主背景：#F6F7F9
- 卡片白：#FFFFFF
- 主文字：#1F2329
- 次文字：#646A73
- 边框：#E5E6EB
- 主按钮：#111111
- 强调金：#C9A45C
- 成功：#00B578
- 警告：#FFB020
- 危险：#F53F3F

### 布局规范
- 左侧导航宽度：220px

## 5. 组件规范

### 5.1 表格页模板
- 顶部：标题 + 主按钮
- 中部：筛选区
- 下方：表格区
- 底部：分页

### 5.2 表单页模板
- 分组卡片式布局
- 单列为主，复杂区域双列
- 保存按钮固定在底部右侧

### 5.3 状态标签
- 启用/上架：绿色
- 草稿/待完善：灰色
- 停用/下架：红色
- 待审核：橙色

## 6. 交互建议

- 删除操作要二次确认
- 上架、发布、停用等关键操作要 toast 提示
- 表单离开未保存时给出提醒
- 上传图片后即时预览
- 推荐位选择作品/艺术家时支持弹窗选择器

## 7. 响应式建议

后台以 PC 端优先，建议设计宽度：
- 最小适配宽度：1280px
- 常规设计宽度：1440px
