# 拾艺局运营后台 PRD v1

## 1. 文档概述

- 项目名称：拾艺局运营后台
- 服务对象：平台运营、内容运营、商品运营、客服、管理员
- 对应前台：拾艺局微信小程序 0.7.0
- 当前阶段：0 到 1 补建后台
- 文档目标：补齐支撑小程序运营、内容录入、用户管理、订单处理的数据管理后台

## 2. 背景与问题

当前仓库已具备以下能力：
- 小程序前台已完成主要页面框架与基础接口联调
- Spring Boot 后端已具备用户、艺术家、作品、订单等主链路接口基础
- 本地数据库已初始化，后端服务已可运行

当前缺失的核心能力：
- 没有独立运营后台页面
- 首页推荐、热门藏品、推荐艺术家等运营内容缺少可视化录入入口
- 用户、艺术家、作品、订单等数据缺少统一管理界面
- 目前依赖 SQL 或临时测试数据，不适合持续运营和交付

因此，必须补建一个可用的运营后台，以支持小程序上线前后的日常运营。

## 3. 产品目标

### 3.1 总目标
建设一个 Web 端运营后台，覆盖小程序当前主链路所需的内容、商品、艺术家、用户、订单管理能力，优先满足 0.7.0 上线与试运营。

### 3.2 阶段目标

#### 第一阶段，MVP
- 支持管理员登录
- 支持首页运营位配置
- 支持艺术家资料管理
- 支持作品管理
- 支持用户管理
- 支持订单管理
- 支持基础数据看板

#### 第二阶段
- 支持证书、托管、物流、分销等运营功能
- 支持角色权限管理
- 支持操作日志
- 支持活动配置

## 4. 用户角色

### 4.1 超级管理员
- 管理全部模块
- 创建运营账号
- 配置角色权限

### 4.2 内容运营
- 管理首页推荐位
- 管理艺术家资料
- 管理作品展示内容

### 4.3 商品运营
- 管理作品状态、价格、库存、上架下架
- 配置热门藏品、升值推荐等专区

### 4.4 客服/订单运营
- 查看用户
- 查看订单
- 更新订单处理状态
- 维护物流信息

## 5. 核心业务范围

本期后台覆盖以下模块：
1. 登录与账号权限
2. 数据看板
3. 首页运营配置
4. 艺术家管理
5. 作品管理
6. 用户管理

8. 系统设置（基础版）

## 7. 功能需求

## 7.1 登录与权限

### 功能目标
为运营人员提供后台登录能力，并控制不同角色可见模块。

### 功能点
- 账号密码登录
- 登录态保持
- 退出登录
- 角色权限控制
- 菜单可见范围控制

### MVP 范围
- 先支持账号密码登录
- 预置两类角色：超级管理员、运营人员

## 7.2 控制台

### 功能目标
帮助运营快速判断平台当前运行状态。

### 关键指标
- 总用户数
- 今日新增用户
- 艺术家数量
- 已上架作品数
- 待支付订单数
- 待发货订单数
- 今日成交金额

### 待办提醒
- 待完善资料的艺术家
- 待上架作品
- 待处理订单

## 7.3 首页运营

### 功能目标
把当前小程序首页“内容为空、推荐位空白”的问题通过后台录入解决。

### 功能点
- 配置首页 Banner
- 配置热门藏品
- 配置正在升值
- 配置推荐艺术家
- 配置首页公告
- 配置排序与生效状态

### 字段建议

#### Banner
- 标题
- 副标题
- 图片
- 跳转类型
- 跳转目标
- 排序
- 是否启用

#### 热门藏品/正在升值/推荐艺术家
- 关联对象 ID
- 展示标题
- 封面图
- 推荐语
- 排序
- 是否启用
- 生效时间

- 个人简介编辑

## 9. 关键流程

### 9.1 首页推荐配置流程
1. 运营登录后台
2. 进入首页运营模块
3. 选择“热门藏品”
4. 从作品库中选择作品
5. 填写推荐语与排序
6. 点击发布
7. 小程序首页展示更新

### 9.2 艺术家入驻内容配置流程
1. 创建艺术家资料
2. 上传头像、封面、简介
3. 关联作品
4. 设置上线状态
5. 小程序艺术家页可见

### 9.3 订单发货流程
1. 订单运营进入订单详情
2. 审核收货信息
3. 填写物流公司与物流单号
4. 更新为已发货
5. 小程序端同步订单状态

## 10. 非功能需求

### 10.1 性能
- 列表页默认响应时间不超过 2 秒
- 支持分页查询

### 10.2 安全
- 登录接口需要鉴权
- 后台接口与小程序接口鉴权隔离
- 关键操作记录操作人

### 10.3 可维护性
- 列表字段、枚举状态统一
- 上传资源统一管理
- 重要操作预留日志能力

## 11. 技术建议

### 11.1 前端后台技术
建议新增独立 Web 后台项目：
- Vue 3
- Vite
- Element Plus 或 Ant Design Vue
- Pinia
- Vue Router
- Axios

### 11.2 后端配套
建议新增后台管理 API 命名空间，例如：
- `/admin/auth/*`
- `/admin/dashboard/*`
- `/admin/artists/*`
- `/admin/artworks/*`
- `/admin/users/*`
- `/admin/orders/*`
- `/admin/operations/*`

### 11.3 数据层补充
建议新增或补充如下表/字段：
- 后台账号表
- 角色权限表
- 首页运营位配置表
- 作品推荐位表
- 操作日志表

## 13. 验收标准

### MVP 验收
- 运营可正常登录后台
- 可在后台录入首页推荐内容
- 可创建并编辑艺术家信息
- 可创建并编辑作品信息
- 可查看用户信息
- 可查看并处理订单状态
- 小程序首页、艺术家页、订单链路可被后台数据支撑

## 14. 风险与注意事项

- 当前小程序与后端接口还在联调阶段，后台字段设计要尽量复用已有模型，避免大面积返工
- 若后端未预留后台专用接口，则需要同步补一轮 admin API
- 首页推荐位和作品、艺术家的关系要提前设计清楚，否则容易出现多处手工维护

## 15. 结论

拾艺局当前已具备小程序前台和业务后端基础，但缺少可交付、可运营、可持续维护的管理后台。为支撑 0.7.0 上线与后续运营，建议立即补建一套后台 MVP，优先完成登录、首页运营、艺术家、作品、用户、订单六大模块。
