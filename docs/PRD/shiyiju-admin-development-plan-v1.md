# 拾艺局运营后台开发文档 v1

## 1. 文档目的

本文档用于明确拾艺局运营后台的开发范围、页面顺序、接口依赖、开发流程与验收规则。

本文件是运营后台的单独开发文档，后续前端、后端、测试、联调都以此为主线推进。

---

## 2. 项目目标

拾艺局运营后台用于支撑平台的：

- 首页运营
- 艺术家管理
- 作品管理
- 用户管理
- 订单管理
- 系统配置
- 艺荐官与增长相关能力（后续扩展）

后台必须同时支持：

1. **PC 端管理后台**
2. **移动端管理后台**

说明：
- PC 端用于日常重度运营、批量操作、数据查看、配置管理
- 移动端用于轻量运营、快速审核、状态查看、紧急处理

---

## 3. 开发总原则

### 3.1 双端原则

每一个运营后台页面都必须考虑：

- PC 端展示与交互
- 移动端展示与交互

禁止只完成 PC 页面而不考虑移动端适配。

最低要求：
- PC 端可完整使用
- 移动端可完成核心操作
- 页面布局具备响应式能力
- 表格类页面在移动端必须转换为卡片式或分组式展示，不能直接强塞桌面表格

### 3.2 页面推进原则

每一个页面必须严格按照以下顺序推进：

1. **开发**
2. **测试**
3. **验收**

只有当前页面完成“开发 + 测试 + 验收”后，才允许进入下一个页面。

禁止：
- 同时铺开多个页面但都未闭环
- 只做骨架不做测试就切下一页
- 前端页面未闭环就盲目扩展后续页面

### 3.3 接口依赖原则

每一个页面在正式开发前，必须先确认该页面依赖接口是否完整。

如果接口不完整：
- 先记录缺口
- 先补后端接口
- 再继续该页面开发

禁止明知接口缺失还直接跳过，导致前端页面出现假闭环。

## 4. 页面开发顺序

当前运营后台页面按以下顺序推进：

### P0
1. 登录页
2. 控制台首页
3. 首页运营页

### P1
4. 艺术家管理页
5. 作品管理页
6. 用户管理页
7. 订单管理页

### P2
8. 系统设置页
9. 移动端快捷工作台
10. 艺荐官中心 / 增长管理（后续扩展）

说明：
- P0 优先解决后台可登录、可看、可运营首页
- P1 解决核心业务管理闭环
- P2 解决配置、移动效率、增长能力

---

## 5. 页面级标准流程

每一个页面都必须执行以下流程。

### 5.1 开发阶段

开发阶段必须完成：

- 页面结构开发
- PC 端视觉与交互完成
- 移动端适配完成
- 页面状态完整（加载、空状态、错误状态、正常状态）
- 关键按钮与交互闭环
- 接口接入

产出物：
- 页面代码
- 页面依赖接口清单
- 当前页面开发说明

### 5.2 测试阶段

测试阶段必须完成：

- 页面渲染测试
- 主要交互测试
- 表单校验测试
- 接口成功 / 失败分支测试
- PC 端测试
- 移动端测试
- 响应式布局测试

产出物：
- 页面测试记录
- 遗留问题清单
- 已修复问题记录

### 5.3 验收阶段

验收阶段必须确认：

- 页面是否满足产品目标
- 页面是否满足运营使用场景
- 页面是否满足 PC + 移动端要求
- 页面是否已经完成关键操作闭环
- 页面是否允许进入下一页开发

只有验收通过，才进入下一页。

## 6. 当前 API 完整度结论

### 6.1 已有后台接口

#### 认证
- `POST /admin/v1/auth/login`
- `GET /admin/v1/auth/me`

#### 控制台
- `GET /admin/v1/dashboard`

#### 首页运营
- `GET /admin/v1/operations`
- `POST /admin/v1/operations`
- `PUT /admin/v1/operations/{id}`

#### 艺术家
- `GET /admin/v1/artists`
- `PUT /admin/v1/artists/{id}/status`

#### 作品
- `GET /admin/v1/artworks`
- `PUT /admin/v1/artworks/{id}/status`

#### 用户
- `GET /admin/v1/users`

#### 订单
- `GET /admin/v1/orders`
- `PUT /admin/v1/orders/{id}/shipment`
- `PUT /admin/v1/orders/{id}/remark`

### 6.2 当前接口缺口

#### 首页运营缺口
- Banner 上传
- 推荐位关联选择
- 推荐位删除
- 推荐位排序
- 模块分组配置

#### 艺术家管理缺口
- 新建
- 编辑
- 详情
- 删除 / 归档
- 排序
- 作品关联

#### 作品管理缺口
- 新建
- 编辑
- 详情
- 图片上传与管理
- 价格规则配置
- 发售方式配置
- 推荐标签配置

#### 用户管理缺口
- 用户详情
- 启用 / 禁用
- 收藏 / 持仓明细
- 权限与角色信息

#### 订单管理缺口
- 订单详情
- 状态流转
- 售后 / 退款
- 物流轨迹
- 更完整筛选条件

#### 系统设置缺口

- 权限接口
- 配置项接口
- 密码重置接口

结论：
当前 admin 后端为**基础版可开发状态**，但不是完整后台接口体系。

## 8. 当前执行策略

从现在开始，运营后台开发执行以下策略：

1. 先以本文档为主线推进
2. 每一页先确认 API 是否可用
3. 每一页必须完成：开发 → 测试 → 验收
4. 每一页完成后，单独汇报
5. 未验收通过，不进入下一页
6. 页面设计必须同时考虑 PC 端和移动端
